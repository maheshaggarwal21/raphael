## 16.4b — atlas bench totals feed the self-use reports (291/291)

The `raph atlas bench` result was ephemeral (printed, then gone). 16.4b makes it
durable and surfaces it in the two self-use reports, all deterministic / zero tokens.

- **Durable event.** `raph atlas bench` now calls `logEvent({ event: 'atlas-bench',
  project, questions, answered, graphTokens, rawTokens, saved, ratio })` after computing
  the totals (commands/atlas.js). Logging is free — the bench itself spends zero model
  tokens — so no gating needed; fires on plain and `--json` runs alike.
- **`raph stats`.** computeStats now collects the LATEST `atlas-bench` event per project
  (latest-by-ts) into `atlas.benches`. renderStats prints an "Atlas leverage" block:
  `<project> : Nx fewer  (raw -> graph over K question(s), date)`, or a "no readable
  candidate files" line when the ratio is null. The empty-log guard now also checks
  benches, so a bench-only brain renders the leverage instead of "nothing recorded yet".
- **`raph report weekly`.** computeWeekly adds an `atlas` block = in-window bench count,
  best ratio, and the latest project/questions. renderWeekly prints one "Atlas leverage"
  line. Windowed like the rest of the report (bench runs outside the N days are ignored).
- Both are pure functions over the events array — no new disk reads, no model, no network.
