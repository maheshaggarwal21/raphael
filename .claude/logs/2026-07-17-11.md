## 16.5 — Obsidian-compatible atlas export (298/298)

Turns the deterministic graph into a self-contained Obsidian vault. New pure module
src/lib/obsidian.js (zero deps, zero tokens, no disk — returns {path,content}):
- renderVault(atlas): one md note per FILE, mirroring the repo path so Obsidian's
  path-form [[wikilinks]] resolve exactly. Each note = forward links (Imports/Tests/
  Calls, call confidence tagged) + BACKREFS (Imported by / Tested by, from reverse
  edges — the value a flat file listing can't give) + Defines/Packages/Error codes +
  a Source line. Plus one note per ERROR CODE (Raised by / Mentioned by = the "where
  does E-SCHEMA come from" awareness view) and an index.md MOC (god-nodes by degree,
  error-code list, the advisory-data framing carried through).
- renderCanvas(atlas): atlas.canvas in JSON Canvas 1.0 (kepano). Top-48 files by
  degree on a deterministic grid (cols = ceil(sqrt n)); type:"file" nodes that open
  the note; edges = import/test among shown files, test edges tinted (color "4").
- Calls deduped by (file,symbol), strongest confidence first (found live: a note had
  the same callee listed 3x from multiple resolved edges).

Command: `raph atlas export [--out <dir>] [--refresh] [--json]` (commands/atlas.js);
default out = brain/atlas/<name>-vault via atomicWrite (mkdirs subfolders). +7 tests
(note-per-file + mirror paths, forward+backref+calls+errors+packages, error notes,
index ranking + canvas link, valid JSON Canvas grid/edges, determinism, maxNotes cap).

16.6a — src/lib/freshness.js (pure, zero deps/tokens) + `raph lint`:
- FRESHNESS (timeless/dated/pointer, warn-only): flags a pinned version (v1.2.3),
  a year, time-relative wording ("currently", "as of", "the latest"), line refs,
  TODO/FIXME. A dated lesson is sometimes legitimately dated -> warn, not error.
- STALENESS (atlas-provable): referencedPaths(lesson) = triggers.paths ∪ path-shaped
  text tokens, but ONLY for file types the atlas indexes (js/ts/py/md/txt...). Any
  such path with no matching atlas file node = STALE; a same-basename-elsewhere = MOVED.
  Capability-check: with no atlas built, the check is skipped and SAID so.
- CONTRADICTION (conservative): lessons sharing >=2 topic terms (keywords + slug
  tokens), with a directional polarity FLIP on a shared term. polarityFor() requires
  the marker within 2 words BEFORE the term and lets negation dominate a co-located
  positive, so "never use floats for money" is NEG about floats, not about money.
  Output labelled "possible" — a review aid, never auto-resolved.
- lintLessons/renderLint aggregate; `raph lint [--project <path>] [--json]`, always exit 0.

Live-caught bug (on the REAL brain): the first pass flagged `.env`, `config`,
`settings`, `package.json` as STALE — but the atlas only indexes code/doc extensions,
so unindexed types + bare hints can NEVER match and would always look stale. Fixed:
staleness only considers atlas-indexed extensions; the real brain now lints clean.

16.6b — retire (mutation), because lint told the user to run `raph retire` and
capability-check forbids referencing a command that doesn't exist:
- retireRefs() in src/lib/review.js (policy in the engine, §14). Irreversible, so it
  REFUSES without --confirmed and prints what would go; with --confirmed it tombstones
  into rejection memory (retired:true, 180-day suppress), removes the active file, logs
  'retired', commits + rebuilds the index. `raph retire <id|slug...> [--reason] --confirmed`.
- Live smoke (sandbox): note -> approve -> retire (refused) -> retire --confirmed (gone);
