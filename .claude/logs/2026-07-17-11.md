16.6a — src/lib/freshness.js (pure, zero deps/tokens) + `raph lint`:
- FRESHNESS (timeless/dated/pointer, warn-only): flags a pinned version (v1.2.3),
  a year, time-relative wording ("currently", "as of", "the latest"), line refs,
  TODO/FIXME. A dated lesson is sometimes legitimately dated -> warn, not error.
- STALENESS (atlas-provable): referencedPaths(lesson) = triggers.paths ∪ path-shaped
  text tokens, but ONLY for file types the atlas indexes (js/ts/py/md/txt...). Any
  such path with no matching atlas file node = STALE; a same-basename-elsewhere = MOVED.
  Capability-check: with no atlas built, the check is skipped and SAID so.
- CONTRADICTION (conservative): lessons sharing >=2 topic terms (keywords + slug
  tokens), with a directional polarity FLIP on a shared term. polarityFor() requires
  the marker within 2 words BEFORE the term and lets negation dominate a co-located
  positive, so "never use floats for money" is NEG about floats, not about money.
  Output labelled "possible" — a review aid, never auto-resolved.
- lintLessons/renderLint aggregate; `raph lint [--project <path>] [--json]`, always exit 0.

Live-caught bug (on the REAL brain): the first pass flagged `.env`, `config`,
`settings`, `package.json` as STALE — but the atlas only indexes code/doc extensions,
so unindexed types + bare hints can NEVER match and would always look stale. Fixed:
staleness only considers atlas-indexed extensions; the real brain now lints clean.
