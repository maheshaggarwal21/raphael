### Web layer (src/lib/web.js)
- GET /api/queue — cards data; refs are candidate FILE NAMES (stable), not queue
  numbers (they shift as items leave).
- GET /api/queue/item?ref= — full frontmatter + body: the console's equivalent of
  `raph show` for the heavyweight review.
- GET /api/stats — same computeStats as `raph stats`.
- POST /api/approve | /api/reject — JSON bodies via readJsonBody (64KB cap,
  fail-closed on bad JSON); mutations go through approveRefs/rejectRefs verbatim.
  Host/Origin/token gate unchanged; a hostile Origin POST is 403 even with the token.
- Page: two tabs. Dashboard = status KPIs, brain table, self-use funnel (stats).
  Queue = cards with severity/category badges; batch approve/reject with optional
  reason for NORMAL candidates only; security/quarantined cards get a lock icon and
  "Review to approve" which renders the ENTIRE candidate (frontmatter JSON + body)
  and only an explicit "I read the full ... candidate" checkbox unlocks a one-item
  "Approve --confirmed" button — the CLI's heavyweight path, mirrored exactly.
- XSS hard line held: every rendered value goes through esc(); CSP still
  default-src 'none' + inline-only; zero external assets; page JS uses no template
  literals (see bug below).
