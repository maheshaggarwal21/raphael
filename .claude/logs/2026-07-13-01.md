## Bugs found and fixed during the session
1. `node --test test/` doesn't resolve a bare directory on Windows → switched to glob `"test/*.test.js"`.
2. Scrubber cascade: the kv-secret rule re-matched our own `<SECRET:...>` placeholders (word "SECRET" + colon looks like a key=value pair) → fixed with a `(?<!<)` lookbehind.
3. Entropy scanner flagged Raphael's own ULIDs (26 random base32 chars) as secrets → exempted `les_/ev_/prj_/mch_` id patterns; safe because the base32 charset can't encode arbitrary secrets.
4. Git Bash `/tmp` maps to `C:\Program Files\Git\` when passed to Node → use real Windows paths for RAPHAEL_HOME in tests/smoke runs. Worth remembering for the eval harness.

## Decisions made
- Strict schema (`additionalProperties: false`) — unknown fields are rejected at the chokepoint, security-preferred.
- Status enum is the superset of both docs' lifecycles: candidate/active/probation/retired/archived/quarantined.
- Imperative voice ("you must...") = quarantine (human must look) but not hard reject; injection-shaped text (deny-list) = hard reject + quarantine.

## Sixth pass (same day) — Phase 5 complete, built solo
- New libs: compile.js (compiled.json builder + hash verification + lazy rebuild),
  stacks.js (manifest-based stack detection, hook-safe), match.js (the deterministic
  weighted scorer with explainable reason strings), inject.js (the recall engine:
  budgets, session state, data envelope, telemetry, fail-open). New commands:
  inject (hook plumbing, always exit 0), search, why, on, off. approve.js now rebuilds
  the index after a successful approve. note.js gained --keywords (so a manual note is
  findable by the per-prompt hook).
- Integrity is structural: compiled.json stores a sha256 per lesson file AND a built_files
  manifest of everything seen at build time; verifyIndex rebuilds on any add/edit/delete/
  tamper, and a skipped-invalid file can't cause a rebuild loop. Every lesson re-passes
  validateLesson() on the way into the index — a hand-edited lesson that stopped validating
  drops out.
- Bug found + fixed (regression): reject.js and search.js both filtered args with
  `i !== flagIdx + 1`. When the flag is absent, indexOf returns -1, so `-1 + 1 = 0`
  silently ate args[0] — i.e. `raph reject 1` with no --reason dropped the ref. Fixed both
  to guard `valueIdx = idx >= 0 ? idx+1 : -1`. Added a regression test for reject without
  --reason. (Same latent bug shape worth watching anywhere we do flag-index math.)
- Honest limitation: cold `raph inject` measured ~300ms on Windows (node startup ~80ms +
  module load/work ~230ms), above the p95<150ms target. Acceptable for SessionStart
  (once/session) and rare prompt fires; logged as a follow-up (warm resident or lighter
  load path). Not blocking.
