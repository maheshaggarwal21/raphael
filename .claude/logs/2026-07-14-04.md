## Phase 7 — project secret guard (`raph init --guard`) — COMPLETE

The last open Phase 7 item: a deterministic pre-commit secret scanner for the USER's own
repos (distinct from the brain's pre-push guard).

### Design calls
- **One definition of "secret."** Refactored `scrub.js` to export `SECRET_RULES` and
  `isHighEntropyToken` (extracted the entropy token test; scrubber behavior unchanged — scrub
  tests still green). The guard scans with the exact patterns the chokepoint uses.
- **High-precision by default.** The named rules (AWS/GitHub/Stripe/Slack keys, PEM private
  keys, JWTs, url-credentials, bearer, `key=secret`) always block. The recall-oriented entropy
  heuristic is opt-in (`--entropy`) — it's noisy on lockfiles/minified assets, the wrong
  default for a commit blocker.
- **Fails open.** Binary/oversized(>1MB)/unreadable files are skipped; a guard must never wedge
  a commit. Named rules only match real secret shapes, so skipping a blob is safe.
- **Scans STAGED content** (`git show :path`) — what's actually about to be committed, not the
  working tree.
- **Non-destructive.** Refuses to clobber a foreign pre-commit hook (`--force` overrides);
  uninstall removes only our own hook. Hook prefers global `raph`, falls back to a baked
  `node <bin>` path so it works before a global install.
