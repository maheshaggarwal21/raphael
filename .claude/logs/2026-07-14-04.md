## Phase 7 — project secret guard (`raph init --guard`) — COMPLETE

The last open Phase 7 item: a deterministic pre-commit secret scanner for the USER's own
repos (distinct from the brain's pre-push guard).

### Design calls
- **One definition of "secret."** Refactored `scrub.js` to export `SECRET_RULES` and
  `isHighEntropyToken` (extracted the entropy token test; scrubber behavior unchanged — scrub
  tests still green). The guard scans with the exact patterns the chokepoint uses.
- **High-precision by default.** The named rules (AWS/GitHub/Stripe/Slack keys, PEM private
  keys, JWTs, url-credentials, bearer, `key=secret`) always block. The recall-oriented entropy
  heuristic is opt-in (`--entropy`) — it's noisy on lockfiles/minified assets, the wrong
  default for a commit blocker.
- **Fails open.** Binary/oversized(>1MB)/unreadable files are skipped; a guard must never wedge
  a commit. Named rules only match real secret shapes, so skipping a blob is safe.
- **Scans STAGED content** (`git show :path`) — what's actually about to be committed, not the
  working tree.
- **Non-destructive.** Refuses to clobber a foreign pre-commit hook (`--force` overrides);
  uninstall removes only our own hook. Hook prefers global `raph`, falls back to a baked
  `node <bin>` path so it works before a global install.

### Files
- `src/lib/scrub.js` — export `SECRET_RULES`, `isHighEntropyToken` (no behavior change).
- `src/lib/guard.js` (new) — `scanText`, `scanFile`, `scanStaged`, git plumbing,
  `installPreCommitHook` / `uninstallPreCommitHook`, `HOOK_MARKER`, `cliBinPath`.
- `src/commands/guard.js` (new) — `raph guard install|uninstall|scan`.
- `src/commands/init.js` — `init(args)`; `--guard` installs the project guard in cwd (never fatal).
- `src/cli.js` — registered `guard`, help text for `guard` + `init --guard`.
- `test/guard.test.js` (new, 12 tests) — scanText line numbers incl. multi-line PEM; entropy
  opt-in; scanFile fail-open; install/uninstall (foreign-hook refusal, force, non-git);
  scanStaged over a real temp git repo.

## Phase 10 — self-use analytics (`raph stats`) — TOOLING COMPLETE

The self-use period is a 2-4 week CALENDAR activity; the code deliverable is the substrate that
records + surfaces the data it produces. Built that.

### Design
- `src/lib/stats.js` = pure aggregation over plain arrays (events + active lessons), fully
  testable without disk. Surfaces the three signals the phase names:
  - TOKEN COST: total/avg tokens per injection, per session, cap hits, date window.
  - RETRIEVAL MISS: active lessons that never appear in an injected event (dead weight, or
    triggers too narrow to be found — the thing to fix).
  - FALSE-FIRE PROXY: lessons firing on prompts with avg score < 4.5 (barely over the 4.0
    gate). Honestly labeled a proxy — a true "unhelpful" signal needs a user feedback channel
    that does not exist yet (logged as a candidate follow-up).
  - Review funnel (approved/rejected/suppressed) always shown, even before any live injection.
- `src/commands/stats.js` = thin: read events.jsonl (skip corrupt lines) + loadIndex(),
  computeStats, render text or `--json`. Read-only, zero network.
- `renderStats` degrades honestly: with approvals but no live injections it shows the review
  funnel and says injection hasn't fired yet (does NOT mislabel all-lessons as retrieval miss).

### Files
- `src/lib/stats.js` (new), `src/commands/stats.js` (new), `src/cli.js` (register + help),
  `test/stats.test.js` (new, 7 tests).
