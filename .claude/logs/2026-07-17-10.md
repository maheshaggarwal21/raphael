## Atlas 16.4 `raph atlas bench` SHIPPED (session 10, 288/288)

The honest number for the "70-80x" claim, measured on our own graph with zero
model tokens.

- benchAtlas(atlas, {questions, tokensForFile}) in atlas.js (pure; file sizing DI'd):
  per question, graph answer = the where() ranked list the agent reads; baseline =
  reading the candidate files WHOLE — but only the files the graph already surfaced,
  so the ratio is conservative/honest, never inflated. Returns per-question rows +
  totals (ratio null when no readable candidates).
- benchQuestions(atlas): auto-derive from the graph — error codes first (the awareness
  use case), then top symbols. renderBench: table + TOTAL + an explicit honest caveat
  (ratio nears 1 on tiny repos / one-small-file answers).
- `raph atlas bench [--questions "a;b;c"] [--json]` (commands/atlas.js): reads candidate
  files from the project root to size the baseline; unreadable/moved files contribute 0.
- Tests (+2): the honest-baseline row math + auto-questions + caveat render; the
  unreadable-files -> ratio null, no-crash path.

LIVE on the raphael repo (10 error-code questions): 174,324 grep+read tokens vs 1,179
graph tokens = 147.9x fewer, 173,145 saved. Per-question 55x (E-FRONTMATTER) to 385x
(E-NOTRANSCRIPTS). This is the deterministic, zero-token confirmation of graphify's
71.5x headline, on our own codebase.
