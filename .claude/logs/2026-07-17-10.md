### Findings (full writeup: docs/atlas-upgrade-plan.md addendum)
- gstack is NOT a Raphael-style distilling brain. It's a ~60-skill library (markdown
  SKILL.md, bun/TS bins) wrapping a dev pipeline (spec->reviews->qa->ship) + safety
  skills + context skills. Memory = two things:
  (1) `learn`: per-project learnings.jsonl, append-only, (key,type) latest-wins dedup,
      numeric confidence 1-10, `/learn prune` = staleness + contradiction check. This is
      the true peer to Raphael's brain — and SIMPLER (no chokepoint, no scrub, no review
      queue, no injection budget).
  (2) `gbrain`: separate optional semantic engine — Postgres/PGLite + vector EMBEDDINGS
      (Voyage/OpenAI) + MCP + Supabase/team sync. The "70-80x"-adjacent piece, but it
      pays for retrieval with a network embeddings API + a vector DB.
- Head-to-head: Raphael is AHEAD on the things that matter for a governed brain —
  safety chokepoint, injection posture, review-before-activation, token accounting,
  determinism (zero network), real eval. gstack's brain is a raw log. Our thesis
  (curation pipeline = the moat) is validated.
