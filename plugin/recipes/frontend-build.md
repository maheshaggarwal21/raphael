# Recipe: Build distinctive UI (not AI slop)

1. raph search "design <keywords>"  — pull the design pack + this project's recorded design decisions so palette/type/spacing stay consistent.
2. Ground it in the subject: name the product, its audience, its one job. Establish a compact token system — 4-6 named hex, 2+ type roles, a spacing scale, ONE signature element.
3. Critique the plan against the generic default BEFORE coding: avoid the cream/serif/terracotta and near-black/acid-green looks, centered layouts, purple gradients, uniform rounded corners, and default Inter. Spend boldness in one place.
4. Build to the plan; hit the floor without announcing it (responsive, keyboard focus, reduced-motion, 4.5:1 contrast, >=44px targets). Copy is design material.
5. Record the chosen design system as a decision (`raph decide`) so the next screen inherits it. Run the free checks; write back any durable design lesson.
