# Recipe: Review a change

1. raph search "<stack> <area>"  — pull lessons that bit here before.
2. Free checks: `git diff --stat`, linter, secret scan — zero tokens, shrink the surface.
3. Read the project map; open only the changed + hot files.
4. Cheap-model sweep of those files; escalate only the top findings to careful review.
5. Report ranked findings with a concrete failure scenario each. Write back a lesson if a real class of bug showed up.
