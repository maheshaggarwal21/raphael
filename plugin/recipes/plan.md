# Recipe: Plan a build from a fuzzy idea

1. raph search "<domain> <product type>"  — pull past scope mistakes for this kind of project first.
2. Planner: iterative inquiry, ONE question at a time (users, core job, success criteria, non-goals, constraints) until the spec is unambiguous. Emit an explicit "NOT in scope" list.
3. Architect: from the spec, design the minimal-but-scalable architecture; emit the Error & Rescue Map (a realistic failure per new codepath + is it handled?) and a "what already exists" note.
4. Hand off the spec + design to the developer/frontend agents. Write back any durable scope or architecture lesson.
