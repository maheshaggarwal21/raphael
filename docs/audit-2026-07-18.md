# Raphael — end-to-end audit & foreign-user test (2026-07-18)

Goal: verify Raphael is trustworthy to hand to renowned tech influencers. Method:
build the **exact npm tarball** that ships (`npm pack`, 146 files), install it into a
**completely isolated global prefix**, and walk the whole journey as a stranger would —
fresh `RAPHAEL_HOME`, no `ANTHROPIC_API_KEY` (subscription-default) — with real samples
and edge cases at every step.

## Verdict

**Ship-ready.** Every advertised feature works as documented for a first-time user. The
core security promises (secret scrubbing, URL/injection rejection, localhost-only console,
containment canaries) all hold under adversarial probing. The audit found **three real
bugs and one doc gap — all fixed, with regression tests** — and confirmed a documented
design boundary in the guard. Test suite: **415/415 green** (was 412; +3 regression tests).

## Bugs found and fixed

1. **Guard missed underscore-fenced env-var secrets** (`src/lib/scrub.js`). The
   `kv-secret` rule used `\b` word boundaries, and `_` is a word char — so the
   archetypal `.env` leak `DB_PASSWORD=…` / `SESSION_SECRET=…` / `AUTH_TOKEN=…` slipped
   past the **default** scan (caught only with the opt-in `--entropy`). Fixed with
   underscore-aware lookarounds; these now block by default. This is the chokepoint's
   single source of truth, so both the brain and the guard tightened together.
   Regression tests added (including a false-positive guard for `DATABASE_HOST=`, `NODE_ENV=`).

2. **`raph contribute <slug>` exported nothing** (`src/commands/contribute.js`). With no
   `--out`, `outIdx` was `-1`, so the arg filter `i !== outIdx + 1` dropped `args[0]` (the
   slug itself). The documented common form silently produced no file. Fixed; command-level
   regression test added (the prior tests only covered the library function).

3. **Console dashboard security text was mode-blind** (`src/lib/web.js`). It always said
   "security always needs a human," which is false in autopilot (security activates via the
   machine curator). Now mode-conditional, matching the accurate Settings-tab copy.

Doc gap: **`raph help` for `arise` omitted `--autopilot`** (the recommended path). Fixed —
the help now leads with `arise --autopilot`.

## Documented design boundary (not a bug)

The guard's high-entropy pass is **opt-in** (`--entropy`), on purpose: enabling it by
default would false-fire on lockfile hashes and block ordinary commits — a worse failure for
a demo. Consequence: a bare high-entropy secret with **no recognizable prefix** and no
secret-keyword (e.g. the 40-char AWS *secret* access key `AWS_SECRET_ACCESS_KEY=…`, where
the keyword isn't the final `=`-adjacent segment) is caught only with `--entropy`. Its
partner — the `AKIA…` access-key *ID* — is a named rule and blocks by default. Security-
conscious users can install the hook with entropy on. Worth a one-line mention in the guard
docs if we want zero surprise.
